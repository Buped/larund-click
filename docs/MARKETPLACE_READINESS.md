# Marketplace Readiness

Larund now has local marketplace foundations:

- local catalog
- package metadata
- package checksum
- requested permission review
- publisher and verified placeholders
- disabled-by-default imports
- MCP and custom API visibility

Not implemented:

- public marketplace
- payments
- remote package trust
- real publisher identity
- signature verification chain

Before a public marketplace, Larund needs signature verification, publisher reputation, package review, malware scanning, revocation, and team approval flows.
